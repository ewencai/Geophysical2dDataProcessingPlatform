# 二维数据批处理平台 — 代码说明（架构）

**版本 V1.0.1**

> 本文说明软件的代码组织与设计约定，供二次开发、命令行调用与维护参考。
> 功能原理与公式见《用户手册》，操作步骤见《操作手册》，安装授权见《安装说明》。

---

## 1. 总体架构

采用**纯函数算子 + 调度引擎 + GUI 外壳**三层分离：

```
App2DProcessor.m        GUI 外壳（App Designer classdef）：功能注册表、输入模式、预览、状态持久化
   │  调用
   ├── +core/           内核引擎：IO 读写、批处理调度、坐标变换、参数默认
   └── +ops/            算子库：27 个算子纯函数 + 共享工具，输入 (G, params) → 输出 G，无副作用
LicenseManager.m        授权模块（RSA-2048 验签，内嵌公钥）
```

设计原则：

- **纯函数算子**：每个 `+ops` 函数形如 `G = opXxx(G, params)`，不读写全局状态、不弹窗、不依赖 GUI，可脱离界面在命令行调用与批量调度。
- **统一网格结构 `G`**：所有算子输入输出同一结构体，空值统一为 `NaN`。
- **注册表驱动 GUI**：功能树、参数控件、输入模式均由 `buildRegistry` 的一张注册表生成，新增算子只需加一条注册项。

## 2. 网格数据结构 `G`

```matlab
G.z        % nrows × ncols 双精度矩阵（第 1 行 = 最北/最高 Y）
G.ncols    % 列数
G.nrows    % 行数
G.xll      % 左下角节点 X 坐标 (m)
G.yll      % 左下角节点 Y 坐标 (m)
G.cellsize % 网格间距 (m)
G.nodata   % 原始空白值标记（写出时复用）
G.name     % 文件名（可选）
```

**坐标约定**：`G.z` 行 1 = 最北，行 `nrows` = 最南；节点配准，最北 Y = `yll + (nrows-1)·cellsize`。

## 3. `+core` 内核（18 个函数）

| 文件 | 职责 |
|---|---|
| `gridRead.m` | 统一读取入口：`.asc`(Esri) / `.grd`,`.dat`(Surfer DSAA 文本) / `.mat`，空值统一 NaN |
| `gridWrite.m` | 写 Esri ArcInfo ASCII（行序北→南） |
| `gridWriteDSAA.m` | 写 Surfer 6 DSAA 文本网格（行序南→北，空白值 `1.70141e38`） |
| `gridWriteMat.m` | 写 `.mat`（结构体变量 `grid`，NaN 无损） |
| `gridWriteXYZ.m` | 写三列 `X Y Z` 文本（节点坐标，NaN 跳过） |
| `scatterRead.m` | 散点读取（`.txt/.csv/.dat/.xyz/.xlsx`，前三列 X Y Z，自动跳表头） |
| `surfer7Read.m` | Surfer 7 二进制网格读取 |
| `surferGrid.m` | 经 COM 调用 Surfer 做高级插值 |
| `batchRun.m` | 单输入批处理调度（失败不中断、进度/取消回调、多输出写盘） |
| `batchRunDual.m` | 双输入批处理（A/B 文件夹按名配对） |
| `batchRunStack.m` | 层堆栈批处理（整栈 + 深度传入算子） |
| `runSingle.m` | 单文件执行一次 |
| `writeResults.m` | 多输出写盘调度（单网格 / struct array / cell array） |
| `coordConvert.m` | Bursa-Wolf 七参数坐标转换 |
| `coordParams.m` | `Par.txt` 参数解析 |
| `listFiles.m` | 通配符文件扫描 |
| `parseDepth.m` | 从文件名解析深度（`_500m` / `delt…to…`） |
| `getOpt.m` | 参数字典默认值取值 |

**批处理容错**：`batchRun*` 对每个文件 `try/catch`，单个出错记录日志继续下一个，不中断整批。

## 4. `+ops` 算子库（27 个算子 + 1 个共享工具）

统一契约：`G = ops.opXxx(G, params)`。频率域算子共享 `wavenum.m`（`|k|` 已含 2π）。

| 类别 | 算子 |
|---|---|
| 网格化 | `opInterp`（14 法：5 MATLAB + 9 Surfer） |
| 网格数学 | `opSmooth`(NaN 感知高斯)、`opDetrend`(多项式去趋势) |
| 位场增强 | `opUpward`、`opVertDeriv`、`opEnhance`(8 法)、`opEuler`(深度反演) |
| 场分离 | `opSeparate`(9 法)、`opSeparateMulti`(逐层+并层多尺度) |
| 网格工具 | `opPullBack`、`opExpand`、`opGridDeriv`、`opIntegrate`、`opConvert` |
| 频谱 | `opFKFilter`、`opRadialSpectrum`(Spector-Grant) |
| 归一化 | `opNormalize`、`opNormalizeToRef`、`opThresholdAnalyze` |
| 网格运算 | `opTimeLapse`、`opSpectralNorm`、`opLayerDiff` |
| 沿层 | `opAlongLayer`、`opLayerStack` |
| 坐标/白化 | `opRotate`、`opCoordTransform`、`opClip` |
| 共享工具 | `wavenum`(波数计算) |

**空白值处理**：频率域算子以有效值均值填充参与 FFT、输出恢复 NaN；空间域算子逐元素跳过 NaN。

**波数约定**（`wavenum.m`）：`|k| = 2π·√(kx²+ky²)`，其中 `kx = 2π·fftAxis(nx)/(nx·d)`，**已含 2π**，求导/延拓/滤波直接乘用。

## 5. GUI 外壳 `App2DProcessor`

- `buildRegistry`：注册表工厂，每条含 `key/name/category/desc/params/fcn/suffix/pattern/reader/inputMode`。
- **输入模式**（`inputMode`）：`grid`(1→1) / `scatter`(散点→网格) / `dualGrid`(2→1~2) / `stack`(N→1 或 N-1)，输入区按此动态重建。
- **多输出预览**：结果为多文件时顶部下拉切换查看。
- **状态持久化**：输入值与路径写入 `appState.json`，下次启动恢复。

## 6. 命令行（Headless）调用

所有算子无需 GUI：

```matlab
addpath('.../project');                       % +core / +ops 在此
G = core.gridRead('bouguer.asc');
G = ops.opUpward(G, struct('height', 1000));
G = ops.opGridDeriv(G, struct('direction','z','order',1));
core.gridWrite(G, 'out.asc');

% 批量
core.batchRun('./in/', '*.asc', './out/', @ops.opSmooth, struct('sigma',3));
```

## 7. 授权模块

- `LicenseManager`：RSA-2048（SHA256withRSA）验签，**仅内嵌公钥**；私钥仅开发端持有。
- 授权码绑定机器指纹（MachineGuid + 主板序列号 SHA-256），含到期签名，离线可验。
- 首启激活，授权文件存 `%APPDATA%\AppLicenseSystem\License.key`。
